# DeerFlow Tutor Agent — E2E 测试用例

> 浏览器端到端走通「AI 老师」核心路径。
> 前置条件：Tutor API 和前端 Vite 均已启动。
>
> 启动方式：
> ```powershell
> # 终端 1：
> cd agent ; .\.venv\Scripts\python.exe -m agent_english.tutor.api
> # 终端 2：
> npm run dev
> ```

## TC-1：词汇学习场景 — 问老师

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1.1 | 点击首页「单词」→ 进入第 1 关 | 进入词汇学习，显示闪卡 |
| 1.2 | 注意页面右下角有「问老师」红色按钮 | 可见 |
| 1.3 | 点击「问老师」 | 底部抽屉滑出；顶部上下文条显示：`vocabulary · 日常问候 · learn · 1/20` |
| 1.4 | 点击快捷问题「怎么记？」 | 消息发送；3–15 秒内收到模型回复（非「假老师」） |
| 1.5 | 确认回复包含中文讲解或记忆技巧 | 回复有实质教学内容 |
| 1.6 | 在输入框输入「再举一个例句」，发送 | 收到一个新的例句，且讲的是同一个词（hello）而非下一张 |
| 1.7 | 手动滑到下一张闪卡（词变为 goodbye） | 界面已切换，但抽屉内讨论内容不变 |
| 1.8 | 再次发送「还有别的用法吗？」 | 老师仍围绕 hello 讲解，不自动切换为 goodbye |
| 1.9 | 点击抽屉顶部「讲当前这题」按钮 | 新消息发送，讨论锚点切换为 goodbye |

**通过条件**：1.4 回复有实质内容；1.6–1.8 多轮不跑偏；1.9 可显式切换。

## TC-2：词汇测验场景 — 防泄题

| 步骤 | 操作 | 预期 |
|------|------|------|
| 2.1 | 完成闪卡学习，进入测验（四选一） | 显示选择题，未作答 |
| 2.2 | 点「问老师」 | 抽屉打开 |
| 2.3 | 输入「这题选什么？」或「正确答案是哪个？」 | 回复不给字母/选项号，只给思路 |
| 2.4 | 检查回复中不含 `A` / `B` / `C` / `D` / `选` + 字母 | 符合防泄题规则 |
| 2.5 | 在题目中随意选择一个答案（对错均可） | 进入 feedback 阶段 |
| 2.6 | 再问「为什么选这个？」 | 此时可以完整讲解答案 |

**通过条件**：2.3–2.4 未作答时不泄标准答案；2.6 已答后可正常讲解。

## TC-3：语法训练（选择题）— 防泄题

| 步骤 | 操作 | 预期 |
|------|------|------|
| 3.1 | 从首页进入「语法」→ 选择任意关卡 | 进入语法训练 |
| 3.2 | 到选择题阶段，未作答 | 显示题目与选项 |
| 3.3 | 问老师「这道语法题选什么？」 | 回复分析思路而非答案 |
| 3.4 | 搜索回复中是否包含 A/B/C/D + 选项号 | 不应该包含 |
| 3.5 | 作答后（对错均可）再问 | 可以给出解析和正确答案 |

**通过条件**：同 TC-2 防泄题。

| 步骤 | 操作 | 预期 |

**通过条件**：收到分析性回复。

## TC-5：多轮会话与记忆

| 步骤 | 操作 | 预期 |
|------|------|------|
| 5.1 | 在词汇关，问老师「这个词怎么记？」 | 回复 |
| 5.2 | 追问「能和别的词一起记吗？」 | 回复与上一个问题不矛盾 |
| 5.3 | 关闭抽屉 | 抽屉收回 |
| 5.4 | 再打开抽屉 | 消息历史保留（可选：MP 阶段实现） |
| 5.5 | 点击「新话题」 | 消息历史清空，上下文条仍显示当前 scene |

**通过条件**：5.2 多轮连贯；5.5 新话题清空前端展示。

## TC-7：错误处理

| 步骤 | 操作 | 预期 |
|------|------|------|
| 7.1 | 关闭 Tutor API 进程 | — |
| 7.2 | 在前端已打开的抽屉中发送消息 | 抽屉内显示错误提示 + 重试按钮 |

| 7.4 | 点击「重试」 | 请求成功，回复正常显示 |

**通过条件**：7.2 不白屏、不崩页面；7.4 可恢复。

## 可用性测试（额外）

- 抽屉打开时不阻塞底层动画/翻卡/答题
- 点击抽屉外区域可关闭抽屉
- 连续快速发送 2 条消息：第二条在前一条未完成时不可发送或排队
- 超长消息（>500 字）输入后发送：应正常截断或提示
